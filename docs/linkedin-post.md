## 🇺🇸 English Version

It was 10pm. I had an idea. My laptop was in another room.

I used to just let it go. Setting up the environment, opening the terminal, finding the right project — sometimes the idea cools down before you can act on it. I use Claude Code as a core part of my development workflow, but it only existed on my PC.

So I decided to change that.

I built a Telegram bot that runs locally on my machine and forwards messages directly to Claude Code via CLI. From my phone, I can create work sessions in specific projects, send prompts with persistent context, receive code back, and even send generated files — like a PDF the Claude just created.

Authentication is via TOTP (rotating code every 30 seconds), so no one else can use the bot even if they find the token.

What changed in my daily workflow: when an idea shows up, I open Telegram, send the message, and Claude is already in the right project context. Zero friction between idea and execution.
